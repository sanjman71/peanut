Step 1:
Using EC2 on Rails, deploy private keys to app and admin users for use with github:
scp -i ~/AWS/id-kdefault .ssh/id* admin@ec2-174-129-118-110.compute-1.amazonaws.com:~/.ssh/
scp -i ~/AWS/id-kdefault .ssh/id* app@ec2-174-129-118-110.compute-1.amazonaws.com:~/.ssh/

Once this is done, on the server get the fingerprint set up so that there isn't a problem first time:
ssh git@github.com

Step 2:
Install gems on the EC2 server:
ssh -i ~/AWS/id-kdefault admin@ec2-174-129-118-110.compute-1.amazonaws.com

Install g++ on the server to build the starling gem below:
sudo aptitude update && sudo aptitude install g++

Must install all the gems.
sudo gem install -v=2.3.2 --no-rdoc --no-ri rails
sudo gem install --no-ri --no-rdoc mbleigh-subdomain-fu
sudo gem install --no-ri --no-rdoc chronic
sudo gem install --no-ri --no-rdoc starling
sudo gem install --no-ri --no-rdoc rubyist-aasm
sudo gem install --no-ri --no-rdoc mislav-will_paginate
sudo gem install --no-ri --no-rdoc -v1.0.6 sanitize
sudo gem install --no-ri --no-rdoc populator
sudo gem install --no-ri --no-rdoc faker
sudo gem install --no-ri --no-rdoc thoughtbot-factory_girl
sudo gem install --no-ri --no-rdoc has_many_polymorphs

Then:
cap ec2onrails:setup
cap ec2onrails:db:create (warning - doesn't seem to indicate if there are problems)
cap deploy:cold

On the server, ran:
rake init:dev_data RAILS_ENV=production
